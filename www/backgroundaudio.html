Background Audio Test Page

There are some, significant, problems with the background audio:

  • A ~25MB MP3 takes between 1 and 2 minutes to load (others have noted 10-20 seconds for a ~4MB MP3).
  • If the MP3 is set to automatically play, when clicking the link, the player doesn't show up until the MP3 is loaded (again, 1-2 minutes).
  • It doesn't work with videos.
  • It, potentially, stops the audio when the home button is pressed (untested, because of Hydration).

So, I would recommend scrapping the background audio. It might be a better option if/when we switch to a native app. It's possible that it will work better if we move away from PhoneGap Build and do all of the compilation, locally. At that point, though, we're working in Xcode. It, certainly, doesn't look promising for this release (which is, probably, already considered late).
